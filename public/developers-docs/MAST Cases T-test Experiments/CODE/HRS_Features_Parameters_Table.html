HRS Score Mapping

Location: Launcher.java lines 1242-1292

Code:

private static int hrsMap(float hrsValue) {
    int map = 0;
    float tolerance = 0.00005F;
    if (hrsValue + tolerance <= 0 || hrsValue - tolerance <= 0) {
        // zero
    } else if (hrsValue + tolerance <= 0.1 || hrsValue - tolerance <= 0.1) {
        map = 1;
    } else if (hrsValue + tolerance <= 0.2 || hrsValue - tolerance <= 0.2) {
        map = 2;
    }
    // ... continues for 0.3, 0.4, 0.5, 0.6, 0.7, 0.8, 0.9, 1.0
    return map;
}

Key Parameter Files to Check

  1. JSON Parameter Files: Each experiment should have a unique JSON file with different parameter values
  2. Parameter File Location: Passed as second argument to the Java program
  3. Expected Format: JSON with keys like "WashoutUpper", "nADCHighPz", "nADCMedPz", "nADCLowPz", etc.

Troubleshooting Identical Values

  1. Check Parameter Files: Verify each experiment uses different JSON parameter files
  2. Validate Parameter Loading: Add logging to see which parameters are actually loaded
  3. Compare Parameter Values: Ensure WashOutUpper and PZ thresholds differ between experiments
  4. Check Alpha Weights: Verify nPZAlphaADC and nPZAlphaDCE values are not identical
  5. Validate Input Data: Ensure different patients are processed for each experiment
